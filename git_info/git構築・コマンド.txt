◾️Gitインストール
・Macの場合
　ターミナルを開き、以下コマンドを実行する
　brew install git

　以下コマンドを実行しバージョンが表示されることを確認し、インストール完了とする
　git --version

◾️ユーザー情報の保存
・以下コマンドを実行し、ユーザー情報を保存する
　git config --global user.name 自分の名前　※Gitに表示されるユーザー名を指定する
　git config --global user.email 自分のメールアドレス　※Gitに登録しているメールアドレスを指定する

※ユーザー情報の保存先のglobal・system・localの違いは以下リンク先を参照
https://qiita.com/shionit/items/fb4a1a30538f8d335b35

・登録したユーザー情報が保存されていることを以下コマンドで確認する
　git config --list

◾️GitHubからクローンする
　※GitHubにクローンするリポジトリが存在することを前提とする
　・git clone リポジトリURL


◾️ブランチの状況確認コマンド
・git status：現在のブランチと変更したファイルの状況を確認
・git branch：作業ディレクトリのローカルブランチ一覧を表示
・git branch -a：リモートブランチを含めたブランチ一覧を表示
・git log：コミット履歴を確認

◾️ブランチ操作
・git checkout -b ブランチ名：新しいブランチの作成と作成したブランチへ切り替える
　※ブランチが存在しない場合のみbオプションは利用可能
・git switch ブランチ名：既存のブランチに切り替える
・git switch -c ブランチ名：ブランチを新規作成して切り替える

◾️コミットまでの操作
・git add --all：現在のすべての変更をステージングへ追加
・git commit -m "コミットメッセージ"：ステージングされているファイルをコミットメッセージと共にコミット
・git restore --staged .：ステージングされた全てのファイルをステージから取り消す（削除しない）

◾️作業ディレクトリ、ステージングの操作
・git restore ファイル名：作業ディレクトリ上の編集内容を取り消す
・git restore --source コミットID ファイル名：指定したファイルを指定したコミットID時点に戻す
・git resotre --staged ファイル名：ステージングしたファイルをステージから取り消す
・git restore --worktree ファイル名：作業ディレクトリ上の編集内容を取り消す

◾️コミットを元に戻す操作
・git reset --soft HEAD~1：直前のコミットを無かったことにする（ファイルは削除しない）
・git reset --mixed HEAD~1：コミット、ステージングしたファイルを作業ディレクトリへ戻す（変更したファイルは削除されない（
・git reset --hard HEAD~1：コミット、ステージング、作業ディレクトリ（変更ファイル）を戻す = 削除する

◾️プッシュする操作
・git push --set-upstream origin ブランチ名：新しいブランチをリモートへプッシュする
・git push origin ブランチ名：既存のリモートブランチへプッシュする

◾️プルする操作
・git pull origin ブランチ名

◾️差分確認
・git diff コミットID..コミットID：2つのコミットIDの差分を確認する
・git diff ブランチ名..ブランチ名：2つのブランチの差分を確認できる
・git diff：編集した内容と直前のコミットの差分を確認できる

◾️スタッシュ（一時退避）
・git stash save：作業ディレクトリのファイルを一時退避させる
・git stash list：スタッシュリストを表示する
・git stash apply stash@{数字}：スタッシュを復活させる。数字の部分にはstash listで確認した番号を指定する
・git stash drop stash@{数字}：スタッシュを削除する。数字の部分にはstash listで確認した番号を指定する
・git stash pop stash@{数字}：スタッシュの復活(apply)と削除(drop)を同時に行う

◾️グレップ
・git grep "検索したい文字列"：リポジトリ内から指定した文字列を検索する
・git grep "検索したい文字列" コミットID or ブランチ名：特定のブランチ、コミットID時点のファイルから指定した文字列を検索する
・git grep -l "検索したい文字列"：指定した文字列が含まれるファイル名を出力する

◾️ブレーム（いつ誰がその行の変更をコミットしたかを調べる）
・git blame ファイル名：指定したファイルの過去の変更履歴（いつ誰のコミットでどの行が変更されたか）を確認できる

◾️リバート
・git revert 打ち消したいコミットID：打ち消したいコミットで行った変更を元に戻す = 変更を元に戻すコミットを行う
※打ち消したいコミットを削除するわけではなく、打ち消したいコミットで行った変更を元に戻して上書き（コミット）する

◾️チェリーピク（別ブランチの特定のコミットのみ現在のブランチに取り込む）
・git cherry-pick コミットID：指定したコミットIDのコミットだけ現在のブランチにコピー、コミットする
・git cherry-pick コミットID..コミットID：指定したコミットID間のコミットを現在のブランチにコピー、コミットする
・git cherry-pick -n コミットID：指定したコミットIDのコミットだけ現在のブランチの作業ディレクトリに変更を加える（コミットは行わない）

◾️リフログ
・git reflog：HEADが動いた履歴（自分が操作した履歴）を見ることができる
　→操作履歴を元にresetで特定の操作のタイミングに戻ることができる

例)
　02f11b7 HEAD@{0}: reset: moving to 02f11b7
　0488e28 HEAD@{1}: merge develop: Merge made by the 'recursive' strategy.
　6978c20 HEAD@{2}: checkout: moving from develop to master
 
　「checkout: moving from develop to master」に戻りたいなら以下コマンドを入力する
　git reset --hard HEAD@{2}：{}内に指定する数字は、reflogの数字を指定する
